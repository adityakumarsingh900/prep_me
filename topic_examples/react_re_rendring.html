React Re-rendering


1. State/Props Change: The re-rendering process starts when there is a change in the component's state or props.

2. Invoke render(): React invokes the render() method of the component. This method returns a new JSX (React elements), which describes what should be displayed on the screen.

3. Generate New Virtual DOM: React then generates a new virtual DOM tree comprising of the returned JSX.

4. Reconciliation: The reconciliation process begins. React compares the new virtual DOM tree with the current one.

5. Diffing Algorithm: React uses a diffing algorithm to efficiently compare the old and new virtual DOMs. It goes through both trees, node by node, to identify any differences.

6. Update Real DOM: If differences (or "diffs") are found, React updates only those parts in the real DOM. This selective updating is more efficient than repainting the entire DOM.

7. Component Updates: If a component's props or state change, React will execute several lifecycle methods, such as componentDidUpdate(). These methods can be used to perform actions when the component updates.

8. Batching: React batches multiple state updates into a single update for performance. This means that if multiple state updates are made in a single event handler, React will batch them into one update and re-render just once.

Remember, React's goal is to minimize updates to the real DOM because such operations are costly. By using a virtual DOM and reconciliation, React can limit the number of updates to the real DOM, leading to better performance.

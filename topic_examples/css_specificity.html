CSS Specificity

Specificity is calculated based on a rule set's component parts:

  1. Inline styles: An inline style is attached directly to the element to be styled. Example: <h1 style="color: #ffffff;">.
  2. IDs: An ID is a unique identifier for the page elements, such as #navbar.
  3. Classes, attributes and pseudo-classes: This category includes .classes, [attributes] and pseudo-classes such as :hover, :focus etc.
  4. Elements and pseudo-elements: This includes element names and pseudo-elements, such as h1, div, ::before and ::after.

When multiple rules apply to the same element and have the same importance (i.e., none are marked as !important), the browser breaks the tie using specificity. The rule with the highest specificity is applied.

If two rules have the same specificity, the one that comes last in the CSS document is the one that will be applied.

Specificity is usually represented by four numbers (a, b, c, d):

  • a counts the number of !important declarations
  • b counts the number of ID selectors
  • c counts the number of class, pseudo-class and attribute selectors
  • d counts the number of element and pseudo-element selectors

For example, the specificity of a rule with one ID selector, one class selector, and one element would be (0, 1, 1, 1).
